前后端分离

为了解决前后端联调的麻烦，比如前端陷入“等接口”的内耗情况，前端同学本地开发完，确认好各个接口已经按照接口文档约定的参数传参后，无需做后端同学的陪练，可以把最新代码发布到某一个开发环境，让后端同学在写完接口后，在开发环境通过页面进行联调。服务端接口如果有问题，后端同学自己修改完重新发布后，再自己从前端页面测试。如果实在需要更改或再增加参数，再去找前端同学。这样前端同学就可以从“改接口-调接口”的循环圈中解脱出来，把更多精力地放在开发工作上。

文档规范

关于接口，不得不提到文档规范，从常见的需求文档到代码中间会经历很多环节，在冗杂的环节中就像传话游戏一样，因此，我认为在整个开发流程中，制订一份规范的书写文档格式尤其重要。其中，写word、markdown文档，更新到公司的交流库上。更好一点的话，使用swagger生成文档。

模拟数据

前后端分离

  • 后台向前台提供API接口,只负责数据的提供和计算，而完全不处理展现
  • 前台通过Http(Ajax)请求获取数据,　在浏览器端动态构建界面显示数据

参考：前后端分离之数据mock_后端mock_梦的遇见OvO的博客-CSDN博客

实用！前后端分离开发之前端模拟数据-腾讯云开发者社区-腾讯云 (tencent.com)

数据mock思路

  • 在client端处数据mock

  • 在server端mock

第一种方式拦截了请求的发出，直接返回mock的数据，而第二种方式请求则真实地发出，只是在server端进行route拦截
前后端分离的方式，就是在传统开发模式中加了一个node层，这样数据mock完全可以在node server端完成，基本思路如下

本地模拟数据

业务代码里直接模拟

在代码里使用假数据，等到获得接口后再注释，灵活性太差，后期太麻烦。

本地请求json

把假数据创建一个json文件，本地请求json文件.

举例：

{
  "Code": 200,
  "Message": "",
  "Data": {
    "items": [{
        "exp1": "fbcc7d9e-9be1",
        "exp2": "xxx",
        "enabled": true,
        "size": 0,
        "createBy": "",
        "createTime": "2019-07-12T02:56:54.17+08:00",
        "updateBy": "",
        "updateTime": "2019-08-06T03:03:37.859+08:00",
        "comment": ""
      }
    ],
    "totalCount": 1
  }
}

在代码里引用：

import jsonData from './data.json';
  • 优点：
    • 数据存储和维护简单，可以在文件中手动编辑。
    • 适用于简单的模拟数据需求。
  • 缺点：
    • 静态数据不支持动态生成，无法模拟实时变化的数据。
    • 需要额外的步骤将 JSON 数据导入到前端项目中。
mock.js(行为有限)

查看mock.js专题文章

  • 优点：

    • 强大的数据生成功能，支持生成各种类型的模拟数据。
    • 能够模拟完整的API接口，包括路由和参数验证。
    • 能够集成到前端代码中，用于前端开发和测试。

  • 缺点：

    • 学习曲线较陡峭，配置相对复杂。
    • 需要在前端项目中引入 Mock.js 库。
faker.js（随机数据生成库）

  • 优点：

    • 可以生成随机和逼真的虚构数据。
    • 适用于填充数据库、生成测试数据等场景。

  • 缺点：

    • 主要用于生成虚构数据，不适用于模拟实际API接口的响应。

  • 示例（使用 Faker.js）：

    javascriptCopy codeconst faker = require('faker');

const data = {
  users: Array.from({ length: 5 }, () => ({
    id: faker.random.number(),
    name: faker.name.findName(),
    email: faker.internet.email(),
  })),
};
API Mocking

  • 优点：

    • 可以模拟完整的API接口行为，包括路由和参数验证。
    • 支持在本地或内部网络上运行。

  • 缺点：

    • 需要安装和配置专门的API Mocking工具，增加了工作量。

    • 不适用于小型项目和简单的模拟数据需求。

举例：JSON Server以及Node+express搭建mock server都属于此类，参考来源

    • 可能不适用于敏感数据，因为数据在外部服务上生成。
